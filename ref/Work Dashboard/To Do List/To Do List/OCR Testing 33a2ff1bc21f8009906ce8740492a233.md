## 3. Technical Trade-offs

### Self-Hosted (DigitalOcean)

- **Pros:** Data privacy; Fixed predictable billing; No vendor lock-in.
- **Cons:** Hardware limitations; Responsible for OCR engine updates (e.g., Tesseract/PaddleOCR optimization);

### Managed API (GCP/Azure)

- **Pros:** Best-in-class accuracy (handwriting support); No server maintenance; High availability ($99.9\%+$ SLA).
- **Cons:** Variable costs can spike with high usage; Requires handling network latency and external API failures.

---

## OCR Deployment Strategy

For the initial launch, a **Self-Hosted VM on DigitalOcean** is the most time-efficient path, as it requires zero code changes in Celery-based architecture. However, it carries a higher fixed monthly cost during low-traffic periods. **Cloud APIs (GCP/Azure)** offer superior scalability and lower entry costs but require a significant upfront development investment (20–40 hours) to integrate and map data.
